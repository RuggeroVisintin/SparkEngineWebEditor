## Core Rules

You MUST stop whenever any of the following conditions is met:

1. Any time a test implementation is changed

2. Before committing any code

3. When changing scripts or pipeline automations


Whenever the implementation becomes complicated, prompt the user if they want an explanation of the reasoning
